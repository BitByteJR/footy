# footy — фронтенд передизайн

## TL;DR

Переделать 3 страницы спорт-инфо сайта (главная / лига / матч) на тёмной теме в стиле букмекерских аппов (Sofascore, FlashScore, 1xBet) для русскоязычной аудитории. Стек строго **FastAPI + Jinja2 + CSS** (без SPA, без сборки). Текущая версия в `templates/` + скрины в `screenshots/`.

---

## Контекст проекта

**footy** — пет-проект, который парсит реальные футбольные данные с football-data.org (топ-6 лиг Европы) и показывает их в букмекерско-стиле UI. Цель — научить DevOps на полном цикле (Docker → CI/CD → k8s → мониторинг), но сейчас фокус на читаемом продуктовом фронте.

**Что есть в БД (живые данные с API):**
- 6 турниров: CL (Champions League), PL (Premier League), PD (La Liga), BL1 (Bundesliga), SA (Serie A), FL1 (Ligue 1)
- 110 команд с эмблемами PNG (URL'ы football-data.org CDN, прозрачные)
- 1941 матч сезона 2025/26 — со счетами для прошедших, датой+временем для будущих
- 132 строки турнирных таблиц (позиция, И/В/Н/П, голы, GD, очки, строка формы "WWLDW")
- 60 топ-бомбардиров (10 на лигу) — имя, клуб, голы, ассисты, матчи

**Чего нет (free tier):**
- ❌ Голы поматчево с именами/минутами — API возвращает `goals: []` для всех матчей
- ❌ Карточки / фолы / владение / xG / удары
- ❌ Лайнапы
- ❌ Фотографии игроков
- ❌ Рыночные стоимости

→ **Дизайн не должен закладываться на отсутствующие данные.** Не делать пустые блоки для голов/xG/владения. Лучше насытить тем что есть: таблица, форма, H2H по счётам, бомбардиры, позиции.

## Стек (строго)

- **Backend:** FastAPI 0.136 + Jinja2Templates. Все шаблоны в одной директории `templates/`. CSS только inline или `<style>` блок (это в Docker контейнере, статика отдельной директорией усложняет).
- **Без HTMX / Alpine / Vue / React** — пользователь явно отверг. Чистый server-side рендер + минимум vanilla JS.
- **Шрифты:** только Google Fonts (preconnect ОК). Бренд-шрифт уже выбран — **Onest** + **JetBrains Mono** для чисел. Поменять можно но не на Inter/Roboto/Arial.
- **Адаптив:** desktop (1280+) и mobile (375-768) обязательно. Tablet вторичен.

## URL-структура

```
GET /                         # главная — топ-матчи + лиги + расписание
GET /league/{code}            # лига — табы Матчи/Таблица/Бомбардиры
   ?tab=matches|table|scorers
GET /match/{match_id}         # матч — анализ перед/после игры
GET /health                   # JSON для мониторинга, фронту не нужен
GET /competitions             # JSON, фронту не нужен
```

## Структура каждой страницы (что должно быть)

### Главная `/`

**Хедер (sticky):**
- Лого «footy» (можно переосмыслить)
- Поиск (placeholder «Команды, лиги, матчи…»)
- Текущая дата справа

**Левый sidebar (240-250px на десктопе):**
- Список 6 лиг: Champions League, Premier League, La Liga, Bundesliga, Serie A, Ligue 1
- Каждая со своим лого + текущим туром (например «Т36»)
- Активная подсвечена лаймовой полосой слева

**Основная область:**
1. **Топ-матчи** — 3-5 крупных карточек с гербами и временем. Что лучше выбрать в качестве «топовых» — гибкость дизайнера (например ЛЧ всегда вперёд + по одному матчу из каждой топ-лиги)
2. **Последние результаты** — лента горизонтальная с компактными парами (текущая реализация ок)
3. **Расписание** — матчи сгруппированы по датам и лигам, с переходом на детальную страницу

### Лига `/league/{code}`

**Hero:**
- Лого лиги крупно (110-150px)
- Название лиги + сезон 2025/26
- 3 stat-карточки: «Клубы 20», «Текущий тур 36», «Лидер: Arsenal · 79 очк»
- Фирменный цвет лиги в градиенте (PL фиолетовый, CL синий, PD красный, BL1 красный, SA зелёный, FL1 navy)

**Tab-навигация:** Матчи (default) | Таблица | Бомбардиры

**В табе «Матчи»:**
- Слева: предстоящие сгруппированы по турам и датам, ниже недавние результаты
- Справа sidebar (sticky) — что-то полезное: например «Сегодня в лиге» с live-секцией, или мини-таблица топ-5 + мини-сетка ближайших туров

**В табе «Таблица»:**
- Полная сортированная таблица: Поз / Клуб / И / В / Н / П / З / П / GD / Очки / Форма
- Цветовые зоны: top-4 (ЛЧ — лайм), 5 (ЛЕ — янтарь), 18-20 (вылет — красный)
- Форма последних 5 — цветные квадраты W/D/L

**В табе «Бомбардиры»:**
- Топ-10 с гербами клубов, голами, ассистами
- Сейчас простой список — можно сделать большую карточку лидера + остальные мелким списком

### Матч `/match/{match_id}`

**Hero:**
- Хлебные крошки: главная · лига · тур
- 3 колонки: home (герб + имя + позиция в таблице) | center (статус + счёт/VS + дата+время) | away
- Фирменный цвет лиги

**Body (две колонки):**

Левая колонка:
- **Голы** — ТОЛЬКО если есть (`{% if match.goals %}`). Хронология с минутами. На free tier пусто — НЕ показывать вообще, заменить на «Форма перед матчем»
- **Форма** — последние 5 матчей каждой команды в виде цветных чипов W/D/L + список с счетами
- **Карточки** — также ТОЛЬКО если есть

Правая колонка:
- **Личные встречи** — все встречи в текущем сезоне + полоса побед/ничьих в цветах. Если 0 — компактная плашка «В этом сезоне не встречались»
- **Позиции в таблице** — мини-таблица где видны обе команды

## Данные (Jinja-контекст)

См. `context/api.py` для точных контекстов каждого роута. Главные модели:

```python
Team:        id, name, short_name, tla, crest_url, competition_id
Match:       id, competition_id, matchday, utc_date (datetime UTC), status,
             home_team_id, away_team_id, home_score, away_score, winner,
             venue, referee, home_team (rel), away_team (rel)
Standing:    competition_id, team_id, position, played, won, drawn, lost,
             goals_for, goals_against, goal_difference, points, form, team
Scorer:      competition_id, player_name, team_id, goals, assists,
             penalties, matches_played, season
Competition: id, code, name, area_name, emblem_url, current_matchday
```

См. `context/models.py` для полных моделей и `context/fdo.py` для API клиента.

## Личные пожелания

- **Букмекерский DNA** — но не клон. Это не казино, это инфо-сайт о футболе. Стиль: серьёзный, плотный, инфо-как-данные, без визуального жира
- **Не AI-slop** — это критично. Глянь референсы: Sofascore Pro, FlashScore Pro, OneFootball, Apple Sports, Goal.com — у них у всех есть характер, не Tailwind-template
- **Mobile-first не нужен**, но мобильная адаптация обязательна (см. league-pl-mobile.png — сейчас sidebar схлопывается в chips сверху, посмотри как лучше)
- **Не бояться плотности**, бояться пустоты. Лучше много информации с хорошей иерархией чем большие пустые карточки
